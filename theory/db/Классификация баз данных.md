
## 1. По модели данных

### **1.1. Реляционные (SQL)**

- Хранят данные в **таблицах (rows × columns)**.
    
- Между таблицами — **отношения (foreign key, join)**.
    
- Основной язык — **SQL**.
    

**Примеры:** PostgreSQL, MySQL, MariaDB, SQLite, SQL Server, Oracle.

**Плюсы:**

- Надёжность, транзакции (`ACID`).
    
- Сложные запросы, `JOIN`, `GROUP BY`, индексы.
    
- Строгая схема, типы данных.
    

**Минусы:**

- Плохо масштабируются горизонтально.
    
- Негибкая схема, тяжело менять структуру данных.
    
- Не всегда быстры на "гибких" данных (например, JSON-документы).
    

**Когда использовать:** финансы, учёт, статистика, чёткие связи — всё, где нужна целостность данных.

---

### **1.2. Нереляционные (NoSQL)**

Общее название для баз, **не следующих модели таблиц и SQL**.  
Делятся по внутренней структуре.

|Подтип|Что хранит|Примеры|
|---|---|---|
|**Key-Value**|ключ → значение|Redis, Memcached, DynamoDB|
|**Document-oriented**|документы (JSON/BSON)|MongoDB, CouchDB, Firebase|
|**Column-oriented**|колонки, а не строки|Cassandra, HBase, ClickHouse|
|**Graph**|узлы и связи|Neo4j, ArangoDB, OrientDB|
|**Time-Series**|временные ряды|InfluxDB, TimescaleDB, Prometheus|

## 4. По назначению (OLTP vs OLAP)

|Тип|Цель|Примеры|
|---|---|---|
|**OLTP (Online Transaction Processing)**|Быстрые транзакции, CRUD, запросы “по пользователям”|PostgreSQL, MySQL|
|**OLAP (Online Analytical Processing)**|Аналитика, агрегации, отчёты, big data|ClickHouse, Snowflake, BigQuery, Redshift|

---

## 5. По хранению данных

|Тип|Описание|
|---|---|
|**In-memory**|данные в RAM (Redis, Memcached)|
|**Disk-based**|классические (PostgreSQL, MongoDB)|
|**Hybrid**|кэш в памяти, storage на диске (RocksDB, LevelDB)|

---

## 6. Ещё пару экзотических видов

|Вид|Примеры|Что делает|
|---|---|---|
|**Search engine DB**|Elasticsearch, OpenSearch|полнотекстовый поиск, индексы, scoring|
|**Ledger DB**|Amazon QLDB, Hyperledger|неизменяемые записи (audit trail)|
|**Vector DB**|Pinecone, Qdrant, Weaviate|хранение embedding-векторов для AI/ML|
|**Multi-model**|ArangoDB, OrientDB|совмещают document + graph + key-value|

---

## 7. Как выбирать тип

|Требование|Тип БД|
|---|---|
|Классические CRUD, отчёты|Relational (PostgreSQL/MySQL)|
|Быстрый кэш|Key-Value (Redis)|
|JSON-данные, гибкость|Document (MongoDB)|
|Аналитика по миллионам строк|Column (ClickHouse)|
|Мониторинг, метрики|Time-Series (InfluxDB, Timescale)|
|Графы и связи|Graph (Neo4j)|
|Поиск по тексту|Search (Elasticsearch)|
