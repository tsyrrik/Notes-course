
## Git Flow
Классическая схема, удобна для долгих циклов релизов.

```

main → стабильная прод-ветка  
develop → интеграционная ветка  
feature/* → разработка новых фич  
release/* → подготовка релиза  
hotfix/* → экстренные исправления

```

- Релизы делаются из `release/*` → `main`
- После релиза `main` мержится обратно в `develop`
- Удобно, если релизы редкие и релиз-менеджмент формальный.

⚠️ Минусы: тяжеловесна, легко “отставить” фичи на месяцы.
