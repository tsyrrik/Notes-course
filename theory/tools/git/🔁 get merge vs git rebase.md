
## Merge
Создаёт новый коммит слияния.

```bash
git merge feature
````

История выглядит “ветвистой”, зато сохраняет контекст всех merge.

**Плюсы:**

- безопасно, не переписывает историю;
    
- удобно для общих веток.
    

**Минусы:**

- история становится громоздкой, много merge-коммитов.
    

---

## Rebase

Переписывает историю, “встраивая” коммиты одной ветки в другую.

```bash
git rebase main
```

**Плюсы:**

- чистая, линейная история;
    
- легко читать `git log`.
    

**Минусы:**

- при ошибках можно потерять коммиты;
    
- не использовать для публичных веток, уже ушедших в общий репозиторий.
    

---

## Совет

- Для фич-веток — `rebase`.
    
- Для интеграции в общие ветки (`main`, `develop`) — `merge`.
