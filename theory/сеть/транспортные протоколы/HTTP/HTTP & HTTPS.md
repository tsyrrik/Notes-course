### Что это

**HTTP (HyperText Transfer Protocol)** — протокол прикладного уровня, по которому браузеры и серверы обмениваются данными.  
Работает поверх **TCP**, по умолчанию на порту **80**.

**HTTPS** — та же штука, но поверх **TLS** (шифрованное соединение), порт **443**.  
То есть HTTPS = HTTP + криптография.

### HTTP — текстовый протокол

Это обычные строки, понятные человеку.  
Запрос состоит из трёх частей:

1. **Стартовая строка**
    
    ```
    GET /users?page=2 HTTP/1.1
    ```
    
    Здесь:
    
    - `GET` — метод
        
    - `/users?page=2` — путь и параметры
        
    - `HTTP/1.1` — версия протокола
        
2. **Заголовки (headers)**
    
    ```
    Host: example.com
    User-Agent: Mozilla/5.0
    Accept: application/json
    Content-Type: application/json
    Content-Length: 72
    ```
    
    Заголовки задают метаданные запроса.
    
3. **Тело (body)**  
    Есть не всегда. Пример у POST:
    
    ```json
    { "name": "Mikhail Savin", "email": "mikhail.d.savin@gmail.com" }
    ```
    

---

### Основные методы

| Метод       | Назначение                          |
| ----------- | ----------------------------------- |
| **GET**     | получить данные                     |
| **POST**    | создать или отправить данные        |
| **PUT**     | заменить существующий ресурс        |
| **PATCH**   | частично обновить                   |
| **DELETE**  | удалить ресурс                      |
| **HEAD**    | получить только заголовки           |
| **OPTIONS** | узнать, какие методы поддерживаются |

Разница между методами по сути только семантическая - не принято гетом создавать записи и постом читать их. Однако сделать так можно, можно в гете отправлять тело, а в посте - квери параметры, потому что запрос в конечном счете просто текст

---

### Коды ответов

|Код|Значение|
|---|---|
|1xx|информационные (редко видишь)|
|2xx|успех (200, 201, 204)|
|3xx|редирект (301, 302, 304)|
|4xx|ошибка клиента (400, 401, 403, 404, 409)|
|5xx|ошибка сервера (500, 502, 504)|

### Отличия HTTP от HTTPS

|Критерий|HTTP|HTTPS|
|---|---|---|
|Шифрование|нет|да (TLS)|
|Порт|80|443|
|Видимость|можно перехватить и прочитать|данные зашифрованы|
|Производительность|быстрее в старых версиях|немного медленнее, но безопасно|
|Использование|локальные тесты, отладка|продакшн, реальные сайты|

HTTPS не “ускоряет”, но делает канал безопасным:  
даже если кто-то перехватит трафик, он не узнает содержимое (ключи обмениваются при TLS-рукопожатии).

---

### Версии протокола

|Версия|Особенности|
|---|---|
|**HTTP/1.0**|каждый запрос = новое соединение|
|**HTTP/1.1**|keep-alive, chunked transfer, стандарт до сих пор|
|**HTTP/2**|бинарный, мультиплексный, быстрее|
|**HTTP/3 (QUIC)**|поверх UDP, без TCP-handshake, ещё быстрее|

---

### Почему HTTPS обязателен

- **Безопасность:** данные шифруются, невозможно перехватить логины/пароли.
    
- **SEO и браузеры:** без HTTPS сайт считается “небезопасным”.
    
- **HTTP/2 и 3 требуют TLS.**
    

---

### Итог

- **HTTP** — это протокол, который определяет, как клиент и сервер обмениваются сообщениями.
    
- **HTTPS** — тот же протокол, но через шифрованный туннель.
    
- Всё, что мы называем REST, GraphQL, RPC — просто едет “поверх” HTTP.
    
- Запрос = стартовая строка + заголовки + (иногда) тело.
    
- Ответ = код + заголовки + тело.
