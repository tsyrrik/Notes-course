команды по типу `man hier`, `git log` и проч могут запускать pager (по умолчанию less)
Ты не должен убивать его `Ctrl+Z`. Это временно "замораживает" процесс и отправляет его в фон.

### Правильный выход

Чтобы нормально выйти из `man`, просто нажми:

```
q
```

(от слова _quit_). Всё, конец страданиям.

### Почему `Ctrl+Z` плохо

`Ctrl+Z` не завершает процесс — он лишь приостанавливает его. Поэтому он остается висеть в фоне в статусе **Stopped**.

### Чтобы увидеть все подвешенные процессы

Введи:

```bash
jobs
```

ты увидишь что-то вроде:

```
[1]+  Stopped                 man hier
```

### Если хочешь его добить окончательно

```bash
kill %1
```

где `%1` — номер из вывода `jobs`.

### Или вернуть его на передний план

```bash
fg %1
```

и потом уже спокойно нажать `q`, как нормальный человек.

Если ты часто попадаешь в эту ситуацию — просто настрой себе рефлекс: `q` вместо `Ctrl+Z`. Это сэкономит тебе нервы и пару жалких процессов-зомби.
