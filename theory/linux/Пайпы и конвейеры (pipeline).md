# Пайпы и конвейеры (pipeline)

`|` — символ пайпа. Он указывает шеллу взять `STDOUT` одного процесса и соединить его с `STDIN` другого процесса. Поскольку большинство утилит читают текст и возвращают текст, их можно свободно комбинировать.

Пример:

```bash
cat source | grep Dog | uniq | sort
```

- Читается файл `source`.
- Входные данные фильтруются по подстроке "Dog" (`grep`).
- Убираются дубли (`uniq`).
- Данные сортируются и выводятся на экран (`sort`).

Unix‑философия, лежащая в основе пайпов:
- Пишите программы, которые делают что-то одно и делают это хорошо.
- Пишите программы, которые бы работали вместе.
- Пишите программы, которые поддерживают текстовые потоки, поскольку это универсальный интерфейс.
