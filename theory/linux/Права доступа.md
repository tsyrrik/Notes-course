# Права доступа

С каждым файлом и директорией ассоциированы права доступа: `r` — чтение, `w` — запись и `x` — исполнение. Права задаются для трёх типов пользователей: владельца (Owner), группы (Group) и остальных (Other).

Пример вывода `ls -l`:

```
-rw-r--r--  evgen  evgen  file.txt
```

Для удобства чтения разделим запись на группы символов: `-` `rw-` `r--` `r--`.
- Первый символ `-` — тип (обычный файл). Для директорий будет `d`.
- Далее три группы по три символа: права владельца, группы и остальных.
  - `rw-` — владелец может читать (r) и писать (w), исполнять — нет (— вместо x).
  - `r--` — у группы только чтение.
  - `r--` — у остальных только чтение.

Удаление файла определяется правами на директорию, в которой он находится: чтобы удалить файл, нужна запись (`w`) в каталог. Сам файл не может описать «право на удаление» — оно берётся из каталога.

Права для директорий:
- `x` — право «прохода» (traverse): позволяет заходить в каталог и обращаться к файлам внутри (при наличии соответствующих прав на сами файлы).
- `r` — право читать список имён внутри каталога (например, `ls`). Чтобы увидеть подробный список (`ls -l`), требуется также `x`, так как нужны метаданные файлов.

Числовая форма прав:
- Используются тройки `rwx` → биты 4 (r), 2 (w), 1 (x).
- Примеры:
  - `drwxr-xr-x` → `755`
  - `-rw-r--r--` → `644`
