# Оболочки командной строки (shell)

Коротко: оболочка — это программа‑интерпретатор команд (REPL), которая читает ввод, запускает процессы и выводит результат. Не путать с «терминалом» — терминал это окно/эмулятор, внутри которого работает оболочка.

Распространённые оболочки: `sh`, `bash`, `zsh`, `fish`. По умолчанию в Ubuntu — `bash`.

## Как узнать и сменить оболочку
- Текущая оболочка: `echo $SHELL` (или посмотреть `echo $0`).
- Доступные оболочки: `cat /etc/shells`.
- Сменить дефолтную: `chsh -s $(which zsh)` (выйдите и войдите снова). Для root — `chsh -s /bin/zsh root`.

## Конфиги и «rc»-файлы
- Bash: интерактивные сессии читают `~/.bashrc`; логин‑сессии — `~/.bash_profile` или `~/.profile`.
- Zsh: основное — `~/.zshrc` (плагины, алиасы, PATH, prompt).
- Где править `PATH`, алиасы, автодополнение — в соответствующем rc‑файле вашей оболочки.

## Типы сессий
- Логин vs не‑логин и интерактивная vs неинтерактивная влияют на то, какие файлы читаются. Для скриптов обычно задают shebang, например:
  ```bash
  #!/usr/bin/env bash
  set -euo pipefail
  ```

## Zsh: с чего начать
- Почему zsh: мощное автодополнение, удобные подсказки, темы/промпт, плагины.
- Статья‑введение: https://habr.com/ru/articles/929352/
- Популярные менеджеры конфигураций: oh‑my‑zsh, zinit, prezto (по желанию).
