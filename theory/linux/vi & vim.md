**vi** — классический консольный текстовый редактор,  
входит **во все Unix-системы и базовые образы Docker**, даже самые минимальные.  
**vim (Vi IMproved)** — его улучшенная версия с подсветкой, буферами, поиском и множеством команд. 
Vim можно докрутить до состояния IDE - подсветка синтаксиса, проваливание в методы и тому подобное.
Вдобавок многие IDE и текстовые редакторы поддерживают раскладку Vim. Использование такой навигации ускоряет тебя в совокупе с слепой печатью, так как не надо переводить руку от мышки к клаве

---

### 1. Почему стоит знать vi

- Устанавливается **по умолчанию везде** — даже если нет `nano` или `less`.
    
- Работает в любой консоли, даже без GUI.
    
- Очень лёгкий и быстрый.
    
- Когда привыкаешь — **редактировать конфиги, коммиты и коды SSH-сессий реально быстрее**, чем в других CLI-редакторах.
    

> vi — как велосипед: тяжело начать,  
> но потом уже не понимаешь, как можно было ходить пешком.

---

### 2. Режимы работы

|Режим|Что делает|
|---|---|
|**Normal**|навигация, команды (по умолчанию при открытии)|
|**Insert**|ввод текста|
|**Command-line**|сохранение, выход, поиск (`:` в начале строки)|

---

### 3. Минимальные команды для выживания

|Действие|Команда|
|---|---|
|Войти в режим ввода|`i`|
|Выйти в normal|`Esc`|
|Сохранить|`:w`|
|Выйти|`:q`|
|Сохранить и выйти|`:wq`|
|Выйти без сохранения|`:q!`|
|Удалить строку|`dd`|
|Отменить действие|`u`|
|Перемещение|`h`, `j`, `k`, `l` (← ↓ ↑ →)|
|Поиск|`/слово` и `n` для перехода к следующему|

---

### 4. Установка (если нет vim)

```bash
sudo apt install vim        # Debian/Ubuntu
sudo dnf install vim        # Fedora/CentOS
brew install vim            # macOS
```

---

### 5. Быстрый пример

```bash
vi /etc/nginx/nginx.conf
# нажми i — редактируй
# Esc :wq — сохрани и выйди
```
