## Вопрос: Разница между git merge и git rebase

## Простой ответ
- merge соединяет ветки через merge-коммит без переписывания истории.
- rebase переписывает историю, «переигрывая» коммиты поверх базовой ветки, чтобы история была линейной.

## Ответ
## git merge
- Объединяет ветки, создавая merge-коммит, не переписывает историю.
- Подходит для публичных веток (main/develop), безопасен в команде.
- Минус: история «ветвистая», появляются merge-коммиты.
```bash
git checkout dev
git merge feature
```

## git rebase
- Переносит коммиты одной ветки поверх другой, переписывает историю (новые хэши).
- Удобно для линейной истории и подготовки feature перед PR.
- Опасно для уже опубликованных веток.
```bash
git checkout feature
git rebase dev
git push --force-with-lease
```

## Когда что
- Merge — если ветка общая/пушнута, нужна сохранённая история.
- Rebase — если ветка локальная, нужна линейность и чистый log.

Главное правило: не rebase публичные ветки.
