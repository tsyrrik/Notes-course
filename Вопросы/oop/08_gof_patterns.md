## Вопрос: Паттерны GoF (Банда четырёх) — кратко

## Простой ответ
- Порождающие — как создавать объекты; структурные — как их соединять; поведенческие — как они общаются.

## Ответ
Простыми словами: 23 классических паттерна из книги GoF, сгруппованы на порождающие, структурные и поведенческие. Выбирайте под задачу, а не “чтобы было”.

## Порождающие (создание объектов)
- Abstract Factory — фабрики фабрик; создать семейство объектов без привязки к конкретным классам.
- Builder — пошаговая сборка сложного объекта, отделяет конструирование от представления.
- Factory Method — делегируем создание подклассам; единая точка создания.
- Prototype — клонирование уже сконфигурированных объектов.
- Singleton — единственный экземпляр (чаще антипаттерн; лучше DI + shared binding).

## Структурные (композиция объектов)
- Adapter — приводит интерфейс к ожидаемому (обёртка над чужим/legacy кодом).
- Bridge — разделяет абстракцию и реализацию, чтобы развивать их независимо.
- Composite — древовидные структуры, работа с деревом как с единым объектом.
- Decorator — добавляет поведение без изменения класса (обёртка).
- Facade — упрощённый вход к сложной подсистеме.
- Flyweight — разделяем общее состояние для экономии памяти (мелкие объекты).
- Proxy — заменитель/посредник для доступа к реальному объекту (ленивая загрузка, кеш, контроль доступа).

## Поведенческие (взаимодействие объектов)
- Chain of Responsibility — цепочка обработчиков до первого, кто справится.
- Command — команда как объект; undo/redo, очереди, логирование действий.
- Interpreter — описать грамматику и интерпретировать выражения.
- Iterator — последовательный обход коллекции без раскрытия её устройства.
- Mediator — централизует коммуникацию между объектами, уменьшая связи.
- Memento — сохранение/восстановление состояния (snapshot).
- Observer — подписка/уведомление об событиях.
- State — поведение зависит от состояния, инкапсулированного в отдельные классы.
- Strategy — взаимозаменяемые алгоритмы под общий интерфейс.
- Template Method — скелет алгоритма в базовом классе, шаги переопределяются в наследниках.
- Visitor — выносит операции над объектами в отдельный посетитель, позволяя добавлять новые операции.
