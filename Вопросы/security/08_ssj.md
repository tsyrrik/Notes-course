# Session Security (сессии)
Простыми словами: защита сессий от угона/подделки/фиксации.

## Простыми словами
- Главный принцип: защищаем куку (HttpOnly/Secure/SameSite), регенерируем сессию после логина и своевременно её истекаем.

## Риски
- Хищение сессии (XSS, перехват cookie).
- Session fixation (навязать жертве чужую сессию).
- Протухшие/неудалённые сессии.

## Защита
- HttpOnly, Secure, SameSite для куки сессии.
- Regenerate session ID после логина/смены прав.
- Короткое TTL + inactivity timeout, принудительный логаут.
- Привязка к устройству/фингерпринту/локации (опционально, осторожно с UX).
- Инвалидация на сервере при logout/смене пароля.
