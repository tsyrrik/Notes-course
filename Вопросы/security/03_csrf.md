# CSRF (Cross-Site Request Forgery)
- Заставляет авторизованного пользователя выполнить запрос без его ведома.
Простыми словами: жертва заходит на чужой сайт, тот тайно дергает ваш бек (где жертва залогинена) и совершает действие (перевод денег, смена почты).

## Простыми словами
- Добавляйте CSRF-токены и SameSite cookies, не доверяйте запросам без вашего токена.

## Защита
- CSRF-токен в формах/заголовках, проверка токена на сервере.
- SameSite/Lax/Strict cookies, `Content-Type` проверки, double-submit cookie.
- Для критичных действий — подтверждение паролем/2FA.
```html
<form method="post">
  <input type="hidden" name="_token" value="<?= $token ?>">
</form>
```
