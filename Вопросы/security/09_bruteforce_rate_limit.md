# Bruteforce и rate limiting
Простыми словами: ограничиваем число попыток (логин, код подтверждения, API) за время, чтобы предотвратить перебор.

## Защита
- Rate limiting по IP/аккаунту/учётке/токену с экспоненциальной задержкой или фиксированным лимитом.
- Капча/дополнительные проверки после N неудач.
- Аппаратный/сетевой уровень: WAF/LB с лимитами.
- Логи + алерты на аномальную активность.
- Не раскрывать детали ошибки («пользователь не найден» vs «неверный пароль»).
