## Вопрос: SQL-инъекция

## Простой ответ
- Никогда не конкатенируем ввод в SQL. Используем подготовленные выражения/плейсхолдеры и белые списки для имен таблиц/полей.

## Ответ
- Подстановка пользовательского ввода в SQL без экранирования/плейсхолдеров.
- Итог: изменение логики запроса, вытаскивание данных, RCE при высоких привилегиях.
Простыми словами: злоумышленник «впихивает» свой SQL в ваш запрос. Если вы конкатенируете строки, вместо WHERE id=1 получите `OR 1=1` и утечку таблицы.

## Защита
- Подготовленные выражения и плейсхолдеры, белые списки идентификаторов.
- Экранирование ненадёжно и не покрывает все случаи.
```php
$stmt = $pdo->prepare('SELECT * FROM users WHERE id = :id');
$stmt->execute(['id' => $id]);
```
