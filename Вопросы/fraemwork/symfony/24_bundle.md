## Вопрос: Bundle в Symfony
Версия: Symfony 8.0 (stable), LTS 7.4; PHP 8.4+ / 8.2+.

## Простой ответ
bundle — «плагин»/пакет с функциональностью (контроллеры, конфиг, ресурсы), который можно подключать/переиспользовать.

## Ответ
- Содержит: код (контроллеры/сервисы), конфиги, ресурсы (шаблоны, переводы), команды.
- Подключение через `config/bundles.php` (Flex делает это автоматически).
- Свои бандлы обычно не нужны для бизнес-кода в modern Symfony — достаточно `src/`.

## Примеры

1. Подключение `DoctrineBundle` через Flex.
2. Собственный Bundle для shared‑модуля в monorepo.
3. Ресурсы бандла: шаблоны/конфиги/команды.

## Доп. теория

1. В современных приложениях чаще обходятся без кастомных Bundle.
2. Bundle полезны для переиспользования между проектами.
