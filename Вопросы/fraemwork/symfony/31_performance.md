## Вопрос: Оптимизация производительности
Версия: Symfony 8.0 (stable), LTS 7.4; PHP 8.4+ / 8.2+.

## Простой ответ
режем лишнее, включаем кеши, смотрим профайлер, оптимизируем SQL и статику.

## Ответ
- OPcache включён и настроен (prod).
- HTTP cache (proxy/HttpCache/Varnish), заголовки `Cache-Control`.
- Application cache (Cache компонент) для тяжёлых операций.
- Минимизировать количество бандлов, автозагрузку только нужного.
- Профайлер/Blackfire для узких мест, логирование медленных запросов DB.
- Оптимизация Doctrine: `fetch joins`, индексы, избегать n+1, пагинация.
- CDN для статики, gzip/brotli на веб-сервере.

## Примеры

1. Включить HTTP‑кэш для публичных страниц.
2. Использовать fetch join в Doctrine для устранения N+1.
3. Включить `opcache` и `APP_ENV=prod`.

## Доп. теория

1. Кэширование даёт максимальный эффект на повторяющихся запросах.
2. Профайлер помогает находить медленные SQL и тяжёлые шаблоны.
