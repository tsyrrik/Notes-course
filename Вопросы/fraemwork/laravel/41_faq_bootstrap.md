## Вопрос: Частые вопросы по инициализации и пайплайну Laravel
Версия: Laravel 12.x (актуальная), 11.x в security-fixes; PHP 8.2–8.4.

## Простой ответ
где настраивать провайдеры, middleware, роуты и как выполнить код после ответа.

## Ответ
- Зачем сервис-провайдеры?  
  Централизованно регистрируют зависимости и настраивают фреймворк/ваши сервисы при запуске приложения.

- Чем отличаются `register()` и `boot()`?  
  `register()` — объявляем биндинги/сервисы в контейнере.  
  `boot()` — используем их (всё уже доступно).

- Где включить/посмотреть middleware?  
  `app/Http/Kernel.php`: глобальные (`$middleware`), групповые (`$middlewareGroups`, web/api) и маршрутные (`$middlewareAliases`).

- Где «подхватывается» роутинг?  
  Провайдеры грузят таблицы маршрутов (`RouteServiceProvider`), сам матчинг делает роутер при обработке запроса.

- Можно ли делать что-то после ответа?  
  Да, через `terminate()` в middleware, очереди/события и другие фоновые механизмы.

## Примеры

1. `composer create-project laravel/laravel` + `php artisan serve`.
2. `npm install && npm run build` для фронта.
3. `php artisan key:generate` при первом запуске.

## Доп. теория

1. Первичный bootstrap включает установку зависимостей и ключа.
2. После изменения env часто нужен `config:clear`.
