## Вопрос: Ленивая загрузка (lazy loading)
Версия: Laravel 12.x (актуальная), 11.x в security-fixes; PHP 8.2–8.4.

## Простой ответ
связанные данные подтягиваются только при первом обращении к связи. Это удобно “по требованию”, но в циклах порождает N+1 запросов.

## Ответ
## Пример
```php
$users = User::all();           // 1 запрос
foreach ($users as $user) {
    echo $user->profile->bio;   // для каждого user ещё по запросу → N+1
}
```

## Как избежать N+1
- Жадная загрузка заранее:
```php
$users = User::with('profile')->get(); // 2 запроса на всё
```
- Ленивая жадная загрузка после выборки:
```php
$users = User::all();
$users->load('profile'); // догружает связи одним запросом
```

Итог: используйте `with`/`load`, когда планируете массово обращаться к связям.

## Примеры

1. `$user->posts` вызывает ленивую загрузку.
2. `User::with('posts')` отключает N+1.
3. `Model::preventLazyLoading()` для dev.

## Доп. теория

1. Lazy loading — причина N+1 без `with()`.
2. В dev можно включить запрет ленивой загрузки.
