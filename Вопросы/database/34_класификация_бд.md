## Вопрос: Классификация БД

## Простой ответ
- Выбирай движок под модель данных и тип нагрузки: CRUD → реляционная, аналитика → колонка, связи → граф, кэш → key-value, поиск → search.

## Ответ
## 1. По модели данных
### 1.1 Реляционные (SQL)
- Таблицы (строки × колонки), связи FK/ JOIN, язык SQL.
- Примеры: PostgreSQL, MySQL/MariaDB, SQLite, SQL Server, Oracle.
- Плюсы: ACID, сложные запросы, индексы, строгая схема.
- Минусы: не гибкая схема, труднее горизонтально масштабировать, не идеальны для “гибких” JSON-документов.
- Когда: финансы, учёт, отчётность, строгие связи и целостность.

### 1.2 Нереляционные (NoSQL)
Подтипы:
- Key-Value: Redis, Memcached, DynamoDB.
- Документные (JSON/BSON): MongoDB, CouchDB, Firebase.
- Колонночные: Cassandra, HBase, ClickHouse.
- Графовые: Neo4j, ArangoDB, OrientDB.
- Time-Series: InfluxDB, TimescaleDB, Prometheus.

## 4. По назначению (OLTP vs OLAP)
- OLTP: быстрые транзакции/CRUD (PostgreSQL, MySQL).  
- OLAP: аналитика/агрегации/big data (ClickHouse, Snowflake, BigQuery, Redshift).

## 5. По хранению
- In-memory: данные в RAM (Redis, Memcached).
- Disk-based: классические на диске (PostgreSQL, MongoDB).
- Hybrid: кэш в памяти, storage на диске (RocksDB, LevelDB).

## 6. Экзотика
- Search engine: Elasticsearch/OpenSearch (полнотекст, scoring).
- Ledger: Amazon QLDB, Hyperledger (неизменяемые записи, аудит).
- Vector DB: Pinecone, Qdrant, Weaviate (embedding/AI).
- Multi-model: ArangoDB, OrientDB (document + graph + key-value).

## 7. Как выбирать
- CRUD/отчёты → Relational (PostgreSQL/MySQL).  
- Быстрый кэш → Key-Value (Redis).  
- Гибкие JSON → Document (MongoDB).  
- Аналитика по миллионам строк → Column (ClickHouse).  
- Метрики/мониторинг → Time-Series (InfluxDB/Timescale).  
- Графовые связи → Graph (Neo4j).  
- Полнотекстовый поиск → Search (Elasticsearch).

## Примеры
```text
OLTP: PostgreSQL / MySQL
OLAP: ClickHouse / BigQuery
Cache: Redis
```

## Доп. теория
- Классификация условна: современные БД часто мульти‑модальные.
- Реальные системы часто используют несколько БД под разные задачи (polyglot).
