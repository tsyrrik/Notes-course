# Миграции
Версионированные изменения схемы БД: создают/изменяют таблицы, индексы, ограничения, seed-данные. Цели: история изменений, согласованность сред, откаты, автоматизация CI/CD. Пример (Laravel Schema): создание таблицы users с полями id/name.

## Простыми словами
- Храним изменения схемы в файлах и прогоняем их одинаково на всех средах.
- Есть `up` и `down`, чтобы откатывать при ошибках.

Пример ручной SQL миграции
```sql
-- up
CREATE TABLE users (
  id serial PRIMARY KEY,
  email text NOT NULL UNIQUE
);
-- down
DROP TABLE users;
```
