# Индексы: зачем и плюсы/минусы
Ускоряют поиск/джойны, `MAX/MIN`, сортировку по префиксам ключа. Типично B-Tree по умолчанию.
Минусы: замедляют INSERT/UPDATE/DELETE, занимают место; бесполезны при низкой селективности.

Пример создания селективного индекса
```sql
CREATE INDEX idx_users_email ON users (email);
-- ускорит WHERE email = 'user@example.com'
```
