# Индексы: зачем и плюсы/минусы
Ускоряют поиск/джойны, `MAX/MIN`, сортировку по префиксам ключа. Типично B-Tree по умолчанию.
Минусы: замедляют INSERT/UPDATE/DELETE, занимают место; бесполезны при низкой селективности.

## Простыми словами
- Индекс — оглавление книги: ускоряет чтение нужных строк.
- Хорош, если фильтруем/сортируем по полю; вреден, если поле почти у всех одинаково или много записей/апдейтов.

Пример создания селективного индекса
```sql
CREATE INDEX idx_users_email ON users (email);
-- ускорит WHERE email = 'user@example.com'
```

Кейс: ускорение сортировки по дате публикации
```sql
CREATE INDEX idx_posts_published_at ON posts (published_at DESC);
-- EXPLAIN покажет Index Scan вместо Seq Scan при ORDER BY published_at LIMIT 10
```
