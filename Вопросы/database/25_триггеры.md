## Вопрос: Триггеры

## Простой ответ
- Авто-хуки в базе: при вставке/обновлении/удалении выполняют ваш код.
- Полезны для аудита/updated_at, но могут скрыть логику и усложнить отладку.

Пример (PostgreSQL)
```sql
CREATE OR REPLACE FUNCTION set_updated_at()
RETURNS trigger AS $$
BEGIN
  NEW.updated_at := now();
  RETURN NEW;
END;$$ LANGUAGE plpgsql;

CREATE TRIGGER users_touch
BEFORE UPDATE ON users
FOR EACH ROW EXECUTE FUNCTION set_updated_at();
```

## Ответ
Код, выполняемый автоматически на `INSERT/UPDATE/DELETE` (BEFORE/AFTER, иногда INSTEAD OF для VIEW). Используются для аудита, бизнес-правил, синхронизации, автозаполнения полей.
