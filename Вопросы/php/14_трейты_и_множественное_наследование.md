## Вопрос: Множественное наследование и трейты
Версия: PHP 8.4; где есть исторические детали — см. по тексту.

## Простой ответ
в PHP нет множественного наследования классов; код переиспользуем через интерфейсы (контракты) и трейты (вмешивание реализаций).

## Ответ
- Один parent-класс, но можно реализовать много интерфейсов.
- Трейты добавляют методы/свойства в классы; разрешение конфликтов через `insteadof`/`as`.

```php
trait Loggable { public function log($m){ echo $m; } }
class Service { use Loggable; }
```
