# Интерфейс vs абстрактный класс
Интерфейс задаёт контракт (методы/константы, PHP 8.1+ — статические методы, 8.4 — свойства без инициализации).
Абстрактный класс может содержать реализацию, свойства, конструктор, финальные методы.
Класс может реализовать несколько интерфейсов, но наследует только один абстрактный класс.

Простыми словами:
- Интерфейс — только контракт (и default-методы), без состояния; даёт множественное «наследование» интерфейсов.
- Абстрактный класс — базовая реализация + состояние/конструктор; один родитель на класс.
- Выбирайте интерфейс, если нужен только контракт; абстрактный класс — если хотите разделить общую реализацию/поле/конструктор.

Пример
```php
interface Logger { public function log(string $msg): void; }

abstract class BaseService {
    public function now(): string { return date(DATE_ATOM); }
    abstract protected function handle(): void;
}

class OrderService extends BaseService implements Logger {
    public function log(string $msg): void { echo $this->now() . ' ' . $msg; }
    protected function handle(): void { /* реализация */ }
}
```
